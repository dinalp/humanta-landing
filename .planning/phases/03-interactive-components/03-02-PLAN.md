  <files>src/components/Navbar.tsx, src/hooks/useActiveSection.ts</files>
  <action>
1. Create `src/hooks/useActiveSection.ts` — a "use client" custom hook:
   - Define `SECTION_IDS = ["what-we-do", "how-it-works", "plans", "faq"]` (matching the section IDs on the page)
   - Use `useState<string>("")` for activeSection
   - In useEffect, create an `IntersectionObserver` with `rootMargin: "-50% 0px -50% 0px"` (triggers when section crosses vertical center)
   - Observe each section by ID; on intersect, set activeSection to `entry.target.id`
   - Cleanup: `observer.disconnect()`
   - Return `activeSection` string
   - Export as named export

2. Rewrite `src/components/Navbar.tsx` to add mobile menu and scroll spy:

   **Imports:** Add `motion`, `AnimatePresence` from `motion/react`; import `useActiveSection` from `@/hooks/useActiveSection`

   **State:** Add `const [menuOpen, setMenuOpen] = useState(false)` alongside existing `scrolled` state. Add `const activeSection = useActiveSection()`.

   **Body scroll lock:** When menuOpen changes to true, set `document.body.style.overflow = "hidden"`; on false, set back to `""`. Use useEffect with menuOpen dependency.

   **Hamburger button:** Add a button visible only on mobile (`md:hidden`) to the right of the logo, before the CTA button. Use an animated hamburger icon with 3 motion.span lines:
   - When closed: 3 horizontal lines (standard hamburger)
   - When open: top line rotates +45deg with y offset, middle line fades to opacity 0, bottom line rotates -45deg with y offset
   - Use `aria-label={menuOpen ? "Close menu" : "Open menu"}`

   **Desktop nav links — active highlighting:** For each nav link, compare `activeSection` with the link's href (strip the #). When active, add visual indicator: `text-white` (instead of `text-white/80`) and an underline via a `motion.div` with `layoutId="nav-indicator"` positioned absolutely below the link text (h-0.5 bg-brand-accent).

   **Mobile menu panel:** Render with AnimatePresence after the nav element:
   - Backdrop: `motion.div` fixed inset-0, bg-black/60, z-40, fades in/out, onClick closes menu
   - Panel: `motion.div` fixed top-0 right-0 bottom-0 w-80 (or w-full on very small screens), bg-brand-dark, z-50, slides from x="100%" to x=0
   - Transition: type "tween", duration 0.3, ease "easeInOut"
   - Inside panel: close button (X icon) at top-right with aria-label="Close menu", then nav links in a vertical flex column with gap-6, text-xl font-heading text-white
   - Each link onClick: call `setMenuOpen(false)` to close menu on navigation
   - Add `role="dialog"` and `aria-modal="true"` on the panel for accessibility
   - Add the "Get in touch" CTA as the last item in the mobile menu

   **Layout adjustments:**
   - The CTA "Get in touch" button should be `hidden md:inline-flex` (hide on mobile, shown on desktop) since mobile uses the hamburger menu
   - The hamburger button should be `md:hidden` (shown on mobile, hidden on desktop)

   IMPORTANT: Keep the existing scroll-based background transition logic unchanged. The hamburger `y` values for the X animation — use y: 8 for a w-6 h-5 container with 3 lines.
