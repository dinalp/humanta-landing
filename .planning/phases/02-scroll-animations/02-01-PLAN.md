  <name>Task 1: Install packages and create animation foundation</name>
  <files>package.json, src/lib/gsap.ts, src/providers/SmoothScrollProvider.tsx, src/hooks/useScrollReveal.ts, src/app/layout.tsx</files>
  <action>
1. Install animation packages:
   ```bash
   npm install gsap @gsap/react lenis
   ```

2. Create `src/lib/gsap.ts` -- centralized GSAP plugin registration:
   - "use client" directive
   - Import gsap from "gsap", ScrollTrigger from "gsap/ScrollTrigger", SplitText from "gsap/SplitText", useGSAP from "@gsap/react"
   - Call `gsap.registerPlugin(ScrollTrigger, SplitText, useGSAP)`
   - Re-export all: `export { gsap, ScrollTrigger, SplitText, useGSAP }`

3. Create `src/providers/SmoothScrollProvider.tsx`:
   - "use client" directive
   - Import ReactLenis from "lenis/react", useLenis from "lenis/react"
   - Import gsap and ScrollTrigger from "@/lib/gsap"
   - Create inner `GsapLenisSync` component that:
     - Calls `useLenis(() => { ScrollTrigger.update(); })` to sync on every scroll event
     - In useEffect: calls `gsap.ticker.lagSmoothing(0)` to prevent frame skipping
   - Export `SmoothScrollProvider` that renders:
     ```tsx
     <ReactLenis root options={{ lerp: 0.1, duration: 1.2, smoothWheel: true, autoRaf: true }}>
       <GsapLenisSync />
       {children}
     </ReactLenis>
     ```
   - Note: Start with `autoRaf: true` (default). If scroll feels laggy or desynced with GSAP during testing, switch to manual RAF driven by GSAP ticker (set `autoRaf: false` and add `gsap.ticker.add((time) => { lenis?.raf(time * 1000); })`)

4. Create `src/hooks/useScrollReveal.ts`:
   - "use client" directive
   - Import useRef from "react", gsap/ScrollTrigger/useGSAP from "@/lib/gsap"
   - Interface `ScrollRevealOptions`: y (default 40), x (default 0), duration (default 0.7), stagger (default 0.12), childSelector (optional string for staggered children), once (default true)
   - Hook creates a ref, uses useGSAP with scope: ref
   - If childSelector provided: targets are `gsap.utils.toArray(childSelector)` scoped to ref, with stagger
   - If no childSelector: target is `ref.current` directly, no stagger
   - ScrollTrigger config: trigger = ref.current, start = "top 80%", toggleActions = once ? "play none none none" : "play none none reverse"
   - Animation: gsap.from(targets, { y, x, opacity: 0, duration, ease: "power2.out", stagger, scrollTrigger })
   - Return ref (typed as `React.RefObject<HTMLElement | null>`)

5. Update `src/app/layout.tsx`:
   - Import SmoothScrollProvider from "@/providers/SmoothScrollProvider"
   - Wrap `{children}` inside body with `<SmoothScrollProvider>{children}</SmoothScrollProvider>`
   - Keep all existing font config, metadata, and class names unchanged
  </action>

<task type="auto">
  <name>Task 2: Hero parallax background and choreographed entrance</name>
  <files>src/components/Hero.tsx</files>
  <action>
Refactor Hero.tsx to add "use client" and implement two animation systems:

1. Add "use client" at top. Add `import { useRef } from "react"`. Import `{ gsap, ScrollTrigger, SplitText, useGSAP }` from `"@/lib/gsap"`.

2. Create a `heroRef` (useRef HTMLElement) on the outer section element.

3. **Parallax background (HERO-02):**
   Add a wrapper div around the background Image for GSAP to target (class "hero-bg-wrapper"). This wrapper should be `absolute inset-0` and contain the Image. The Image itself keeps `fill` and `priority`.

   Inside useGSAP (scope: heroRef):
   - Use `ScrollTrigger.matchMedia` to only apply on desktop `"(min-width: 768px)"`:
     - `gsap.to(".hero-bg-wrapper", { yPercent: 20, ease: "none", scrollTrigger: { trigger: heroRef.current, start: "top top", end: "bottom top", scrub: true } })`
   - This makes the hero background move ~20% slower than scroll, creating subtle depth.
   - The parent section already has `overflow-hidden` -- confirm it stays to clip the parallax overflow.

4. **Choreographed entrance (HERO-03):**
   Add CSS classes to target elements:
   - Headline h1: add class "hero-headline"
   - Description p: add class "hero-subtext"
   - CTA link: add class "hero-cta"
   - Tagline p: add class "hero-tagline"

   Set initial opacity to 0 on tagline, subtext, and CTA via inline style `style={{ opacity: 0 }}` to prevent flash of unstyled content before GSAP runs. Do NOT set opacity 0 on headline -- SplitText handles that per-word.

   Inside the same useGSAP callback (after parallax setup):
   - Create SplitText: `const split = SplitText.create(".hero-headline", { type: "words" })`
   - Build a timeline: `const tl = gsap.timeline({ defaults: { ease: "power3.out" } })`
   - Sequence:
     a. `.from(split.words, { y: 60, opacity: 0, duration: 0.8, stagger: 0.08 })`
     b. `.from(".hero-tagline", { y: 20, opacity: 0, duration: 0.5 }, "-=0.4")` -- tagline fades in overlapping with headline end
     c. `.from(".hero-subtext", { y: 30, opacity: 0, duration: 0.6 }, "-=0.2")`
     d. `.from(".hero-cta", { y: 20, opacity: 0, duration: 0.5 }, "-=0.2")`

   This creates the choreographed flow: words stagger in -> tagline appears -> subtext fades up -> CTA button arrives last.

5. **Important implementation notes:**
   - Do NOT animate the section element itself (it may be used for scroll calculations). Animate children only.
   - The `useGSAP` hook with `scope: heroRef` automatically handles cleanup of SplitText and ScrollTrigger instances.
   - Keep the existing HERO_CONTENT import and all content/structure unchanged. Only add refs, classes, and the useGSAP block.
