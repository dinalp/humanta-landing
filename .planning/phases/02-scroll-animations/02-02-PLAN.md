This is the most complex animation in Phase 2. Refactor Story.tsx from stacked layout to a pinned scroll animation on desktop while keeping stacked layout on mobile.

1. Add "use client" at top. Import `useRef` from "react". Import `{ gsap, ScrollTrigger, useGSAP }` from `"@/lib/gsap"`.

2. **Desktop structure (md and above):**
   The outer section must allow ScrollTrigger to pin it. Structure:
   ```
   <section ref={containerRef} id="what-we-do" className="bg-brand-dark">
     {/* Mobile: stacked panels (visible below md) */}
     <div className="md:hidden py-24 ...">
       {STORY_PANELS.map(...stacked layout with classes for scroll reveals...)}
     </div>

     {/* Desktop: pinned container (hidden below md) */}
     <div className="hidden md:block relative h-screen" ref={pinContainerRef}>
       {STORY_PANELS.map((panel, index) => (
         <div key={panel.title} className="story-panel absolute inset-0 flex items-center" style={{ opacity: index === 0 ? 1 : 0 }}>
           <div className="max-w-6xl mx-auto px-6 flex gap-16 items-center">
             {/* Text side */}
             <div className="w-1/2 panel-text">
               <h2>...</h2>
               <p><HighlightedText .../></p>
             </div>
             {/* Image side */}
             <div className="w-1/2 relative overflow-hidden rounded-2xl aspect-[3/2]">
               <div className="panel-image w-full h-full" style={{ clipPath: index === 0 ? "circle(100% at 50% 50%)" : "circle(0% at 50% 50%)" }}>
                 <Image src={panel.image} alt={panel.imageAlt} fill className="object-cover" sizes="50vw" />
               </div>
             </div>
           </div>
         </div>
       ))}
     </div>
   </section>
   ```

3. **Desktop GSAP animation (inside useGSAP, scope: containerRef):**
   Use `ScrollTrigger.matchMedia` for `"(min-width: 768px)"`:

   - Get panels: `const panels = gsap.utils.toArray<HTMLElement>(".story-panel")`
   - Build a timeline with ScrollTrigger pinning:
     ```
     const tl = gsap.timeline({
       scrollTrigger: {
         trigger: pinContainerRef.current,
         pin: true,
         scrub: 1,
         start: "top top",
         end: "+=300%",  // 3x viewport for 3 panels
       }
     });
     ```
   - For panels at index 1 and 2 (panel 0 is already visible):
     - Fade out previous panel text: `tl.to(panels[i-1].querySelector(".panel-text"), { opacity: 0, duration: 0.3 })`
     - Fade in current panel (the whole panel): `tl.to(panels[i], { opacity: 1, duration: 0.3 }, "<")`
     - Fade in current panel text: `tl.from(panels[i].querySelector(".panel-text")!, { opacity: 0, y: 30, duration: 0.3 }, "<0.1")`
     - Clip-path reveal on image: `tl.fromTo(panels[i].querySelector(".panel-image")!, { clipPath: "circle(0% at 50% 50%)" }, { clipPath: "circle(100% at 50% 50%)", duration: 0.5 }, "<")`
     - Add a small pause between panels: `tl.to({}, { duration: 0.2 })`

   **CRITICAL anti-pattern:** Do NOT animate transform/position on the pinned element (pinContainerRef). Only animate children (the `.story-panel` divs and their children).

4. **Mobile animation (below md):**
   Inside the same matchMedia block, add `"(max-width: 767px)"`:
   - Select all mobile panels with a class like `.story-panel-mobile`
   - Apply simple gsap.from on each: `{ y: 40, opacity: 0, duration: 0.7, ease: "power2.out", scrollTrigger: { trigger: el, start: "top 85%" } }`
   - Mobile panels should have `style={{ opacity: 0 }}` initially to prevent flash, or use the class approach

5. **Keep HighlightedText** helper component unchanged. Keep STORY_PANELS import unchanged.

6. After the timeline is set up, add a `ScrollTrigger.refresh()` call after a short delay to handle any layout recalculation:
   ```
   // Refresh after images load
   ScrollTrigger.refresh();
   ```
